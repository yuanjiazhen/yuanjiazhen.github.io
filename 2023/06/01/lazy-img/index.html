网页性能优化-图片懒加载

Jia

2023-06-01 21:33:34

背景

当一次性加载大量图片时，会发起大量的网络请求，而这些请求几乎是并发的（不同浏览器对并发请求策略不同），就会导致资源加载时间过长，影响页面加载速度，影响用户体验，于是就出现图片懒加载方案优化性能

概念

图片懒加载简单来说就是按需加载资源，当页面中图片进入用户视图时才去加载资源，也可按需求在图片具体出现在视图 X 距离时就提前加载资源

实现方案

以下方案均在 React 框架下实现的

方案一：JS计算

js 计算核心就是通过比较图片距离视窗高度和视窗高度来判断图片是否进入视图，从而决定是否加载资源
实现原理：

  1. 预先生成所有的<img data-src="https://xxx" src={loadingImgUrl} alt="alt"/>
    • data-src：存放图片资源地址，之后用于 src 替换
    • loadingImgUrl： 预先用 loading 图片占位，多个<img/>使用的是同一个 loading 图片，浏览器只会加载一次
  2. 页面挂载之后，注册滚动事件监听（也可按需求注册屏幕尺寸变化、屏幕旋转等会影响图片位置的事件），监听图片是否进入视窗
  3. 通过 Element.getBoundingClientRect() 获取目标元素的坐标信息
  4. 比较目标元素的高度是否小于或等于 window.innerHeight，小于的话代表图片进入视窗
  5. 图片进入视窗，用 data-src 值替换 src 值
  6. 当页面卸载或所有图片加载完移除监听
const imgs = ['https://xx-1', 'https://xx-2', ...]
function ImgJs() {
  useEffect(() => {
    lazyLoadImg()
    document.addEventListener('scroll', lazyLoadImg)
    return () => {
      document.removeEventListener('scroll', lazyLoadImg)
    }
  }, [])

  const isInView = (element) => {
    const viewHeight = window.innerHeight || document.documentElement.clientHeight
    const { top, left } = element.getBoundingClientRect()
    const isFirstInView = top >= 0 && top <= viewHeight && left >= 0
    return isFirstInView
  }

  // 通过元素与视图窗口高度对比，判断元素是否出现在视图，是否需要加载
  const lazyLoadImg = dedounce(() => {
    const allImg = document.querySelectorAll('.lazy')

    // 浏览器加载同一资源地址（loading）不会重复请求的，只会请求一次
    allImg.forEach(img => {
      if (isInView(img)) {
        // 图片出现在可视区域，加载资源
        img.src = img.dataset.src
        img.classList.remove('lazy')
        delete img.dataset.src  // 删除 data-src
      }
    })
    if (!allImg.length) {
      // 所有图片加载完，移除监听
      document.removeEventListener('scroll', lazyLoadImg)
    }
  }, 100)

  return (
    <div>
      {imgs.map((item, index) => (
        <img key={index} data-src={item} src={loading} alt="alt" className="lazy" />
      ))}
    </div>
  );
}

方案二：IntersectionObserver

IntersectionObserver核心就是观察目标元素是否进入视窗来判断图片是否加载
注意 IntersectionObserver 不兼容 IE，详细兼容表如下

实现原理：

  1. 预先生成所有的<img data-src="https://xxx" src={loadingImgUrl} alt="alt" />
  2. 页面挂载之后，注册观察对象 imgObserver
  3. 通过观察元素 isIntersecting 比较目判断是否进入视窗
  4. 图片进入视窗，用 data-src 值替换 src 值，移除观察
function ImgJs() {
  useEffect(() => {
    lazyLoadImg()
  }, [])

  // 通过 IntersectionObserver 判断元素是否出现在视图，是否需要加载
  const lazyLoadImg = dedounce(() => {
    const allImg = document.querySelectorAll('.lazy')
    const imgObserver = new IntersectionObserver((entries) => {
      entries.forEach(item => {
        if (item.isIntersecting) {
          // 元素出现在视图中
          item.target.src = item.target.dataset.src
          item.target.classList.remove('lazy')
          delete item.target.dataset.src
          imgObserver.unobserve(item.target)  // 移除监听
        }
      })
    })
    // 注册所有目标元素的监听
    allImg.forEach(item => {
      imgObserver.observe(item)
    })
  }, 500)

  return (
    <div>
      {imgs.map((item, index) => (
        <img key={index} data-src={item} src={loading} alt="alt" className="lazy" />
      ))}
    </div>
  );
}

当图片为背景图

当图片为背景图，判断图片进入视图用方案一或方案二都可以，只是在图片资源替换上不通过，如下：

if (item.isIntersecting) {
  // 元素出现在视图中
  item.target.classList.remove('lazy-bg') // 移除class
  imgObserver.unobserve(item.target)  // 移除监听
}

<div>
  {imgs.map((item, index) => (
    <div key={index} style={{ background: `url(${item})` }} className="img lazy-bg" />
  ))}
</div>

  background: url('https://loading') !important;
}
