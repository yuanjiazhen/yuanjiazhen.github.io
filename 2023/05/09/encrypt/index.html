常见加密算法

Jia

2023-05-09 12:17:34

安全和算法

  1. 互联网中传输数据的 4 个问题

    • 窃听：A 和 B 通信时，消息被 X 偷看
    • 假冒：A 和 B 通信时，X 冒充了 A 或 B
    • 篡改：A 和 B 通信时，消息内容被 X 修改了
    • 事后否认：A 向 B 发送了消息，但 A 否认了（会导致互联网上签署的商业交易或合同无法成立）

  2. 解决互联网中传输数据的 4 个问题的方法

    • 窃听：用加密技术解决
    • 假冒：用消息认证码/数字签名解决
    • 篡改：用消息认证码/数字签名解决
    • 事后否认：用数字签名解决
    • 备注：数字签名存在“无法确认公开密钥制作者”问题，可用数字证书解决

加密的基础知识

  1. 对计算机来说，数据是一串有意义的数字罗列，密文只不过是计算机无法理解无规律的数字罗列
  2. 加密就是数据经过某种运算，变成计算机无法理解的数的过程

加密方式

哈希函数

  1. 哈希函数把数据转换成固定长度无规律的数值（多用十六进制表示）
  2. 如何输入数据相同，输出的哈希值也相同
  3. 哈希冲突：输入数据不用，但输出哈希值相同
  4. 哈希值不可逆：不可以从哈希值反推到原来数值；可服务器存储用户账号密码
  5. 哈希代表算法：MD5（有安全隐患）、SHA-1（有安全隐患）、SHA-2（广泛）

共享密钥加密

  1. 共享密钥加密（对称加密）：加密和解密都使用相同密钥
  2. 代表算法：凯撒密码、AES（广泛）、DES、动态口令等
  3. 弊端-窃听：共享密钥是要在网络中传给对方，传输时存在被 X 盗取风险，从而导致 A B 通信消息被 X 盗取并解密

公开密钥加密

  1. 公开密钥加密（非对称加密）：加密和解密都使用不同密钥，适用于多人传输数据，但加密解密耗时，不适用持续发送零散数据
  2. 代表算法：RSA 算法（广泛）、椭圆曲线加密算法
  3. AB 双方通信，需由接口方 B 生成公开密钥和私有密钥，然后将公钥传输给发起方 A，A 将密文（消息 + 公钥 = 密文）传输给 B，B 用私钥解密从而得到消息内容
  4. 弊端-中间人攻击：AB 双方通信时，X 生成自己的公钥 Px 和 Sx，将 Px 给 A，A 无法确认 Px 是否是 B 的，将密文（消息 + Px = 密文）发送给 B 过程中，密文被 X 窃取，并用 Sx 解密，从而得到消息内容

混合加密

  1. 背景：为了解决共享密钥加密无法安全传输密钥问题和公开密钥加密加解密耗时问题，从而结合两种方法以实现互补从而产生混合加密
  2. 方法：共享密钥加密消息，公开密钥加密密钥（AB 通信，B 生成 Pb 和 Sb，将 Pb 给 A ，A 拿到 Pb 加密自己的密钥，传输给 B，B 用 Sb 解 A 密钥得到密钥）

消息认证码

  1. 用于实现认证和检测篡改
  2. AB 通信，A 生成制作消息认证码的密钥，安全的传输给 B。A 用密文和密钥生成一个值（消息认证码 MAC）, A 将密文和 MAC 传输给 B。B 拿到密文 和 MAC ，用密钥 + 密文生成新的 MAC 和 A 传输过来的 MAC 对比是否相等，相等则密文没被篡改，可用密钥解密密文得到消息，否则密文已篡改，需重新向 A 请求数据。
  3. 计算 MAC 的算法：HMAC（广泛）、OMAC、CMAC
  4. 弊端：AB 方持有一样的密钥，无法确认 MAC 是由谁生成的，会带来事后否认问题

数字签名

  1. 用于实现认证和检测篡改、预防事后否认（只有发送者才能生成数字签名）功能
  2. AB 通信，A 需准备消息 + 公钥 Pa + 私钥 Sa，将 Pa 发给 B，A 用数字签名（Sa + 消息） + 密文发送给 B，B 用 Pa 解数字签名得到消息和收到的消息比对是否一致，一致则消息未被篡改
  3. 数字签名使用公开密钥加密生成
  4. 公开密钥加密加解密比较耗时，为节省运算时间，一般先求消息哈希值，再对哈希值加密作为数字签名使用
  5. 弊端：会带来假冒问题，因为 B 无法确认是不是 X 冒充 A的

数字证书

  1. 用来解决数字签名的弊端
  2. 区别于数字签名：
    • A 将 Pa 发给 B 时，需向认证中心申请发行证书，证明 Pa 是 A 的：A 将 Pa + 包含邮箱的个人资料发给认证中心，认证中心对收到的资料确认是 A 的资料后，用认证中心的私钥 Sc + A 资料生成数字签名，认证中心将数字签名 + A 资料打包成一个文件（数字证书）发回给 A，A 用这个文件作为 AB 通信的数字证书
    • B 收到数字证书需确认资料是 A 的，然后向认证中心获取公开密钥 Ps，B 用 Ps 认证数字签名是否是认证中心发出的
    • B 确认了证书是认证中心发出的，邮箱地址是 A 的后，就从证书中取出 Pa，进行通信
