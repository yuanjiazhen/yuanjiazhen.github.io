React 批量更新

Jia

2023-03-10 20:52:43

概念

批量更新是指 React 将多次 state 更新进行合并处理，最终只进行一次渲染，以获得更好的性能，如以下：

function App() {
  const [name, setName] = useState('')
  const [age, setAge] = useState(0)

  const handleClick = () => {
    // setName、setAge 两次更新最终会批量处理，只触发一次更新渲染
    setName('xiaoming')
    setAge(18)
  }
  return (
    <button onClick={handleClick}>{name}-{age}</button>
  )
}

作用

  1. 可避免不必要渲染，提高组件渲染性能

版本差异

  1. react 17 在浏览器事件中（如点击事件）才会触发批量更新，外部事件不会触发批量更新操作，如下：
  2. React 17 只在 React 事件处理函数中执行过程中进行批量更新，默认情况下，不处理 promises、setTimeout、原生事件处理函数等
  3. React 18 使用 createRoot(使用 render 和 v17 一致)会对所有更新都自动进行批量处理，包括 promises、setTimeout、原生事件处理函数等
function App() {
  const [name, setName] = useState('')
  const [age, setAge] = useState(0)

  const handleClick = () => {
    // React 17 fetchSomething 里面更新不会批量处理，会触发两次更新渲染
    // React 18 使用 createRoot fetchSomething 里面更新会批量处理，只触发一次更新渲染
    fetchSomething().then(() => {
      setName('xiaoming')
      setAge(18)
    })
  }
  return (
    <button onClick={handleClick}>{name}-{age}</button>
  )
}

注意：React 18 自动批量更新对 Hook 通常没影响，但对 Class 会存在以下问题：Class 中是可以在两次更新中读取 state 的，如：

import React from 'react';
class App extends React.Component {
  constructor() {
    super();
    this.state = {
      count: 0,
      flag: false
    }
  }
  render() {
    handleClick = () => {
      setTimeout(() => {
        this.setState(({ count }) => ({ count: count + 1 }));
        // 当需要在两次 setState 中读取 count
        // React 17 class：{ count: 1, flag: false }
        // React 18 class：{ count: 0, flag: false }
        // 如果需要获取到 count 为 1，可用 flushSync 对上面的 setState 进行更新
        console.log(this.state);
        this.setState(({ flag }) => ({ flag: !flag }));
      }, 0);
    };
    return (
      <button onClick={handleClick}>{name}-{age}</button>
    )
  }
}

强制批量更新

React 17 如果想在不支持批量更新事件中强制批量更新，可以使用 unstable_batchedUpdates，虽然 unstable_batchedUpdates 在 React 18 可能用不到，但官方还未在 18 中废除这个 API，当主流库不再依赖于它之后，可能在未来的某个版本会删除它

import { unstable_batchedUpdates } from 'react-dom'
function App() {
  const [name, setName] = useState('')
  const [age, setAge] = useState(0)

  const handleClick = () => {
    setTimeout(() => {
      unstable_batchedUpdates(() => {
        setName('xiaoming')// 触发一次 render
        setAge(18)// 触发一次 render
      })
    }, 0)
  }
  return (
    <button onClick={handleClick}>{name}-{age}</button>
  )
}

强制不批量更新

通常批量更新是安全的，但有些情况可能依赖于在状态更改后立即从 DOM 中读取某些内容，这时可以使用 ReactDOM.flushSync()强制不批量更新

import { flushSync } from 'react-dom'
function App() {
  const [name, setName] = useState('')
  const [age, setAge] = useState(0)

  const handleClick = () => {
    flushSync(() => {
      // 触发一次 render
      setName('xiaoming')
    })
    // 当 setAge 依赖 setName 的结果，可以使用 flushSync 来避免批量更新
    flushSync(() => {
      if (name === 'xiaoming') {
        // 触发一次 render
        setAge(18)
      }
    })
  }
  return (
    <button onClick={handleClick}>{name}-{age}</button>
  )
}
