Jia

简单Diff

由于操作 DOM 的性能开销是比较大的，所以就出现了 Diff 算法。Diff 算法通过对比新旧 vnode 节点，找出需更新的节点，定向更新。使用 Diff 比较两个节点的时候，只会进行同层比较

以下面新旧节点为例

// 旧 vnode
const oldVNode = {
  type: 'div',
  children: [
    { type: 'p', children: '1' },
    { type: 'p', children: '2' },
    { type: 'p', children: '3' }
  ]
}

// 新 vnode
const newVNode = {
  type: 'div',
  children: [
    { type: 'p', children: '4' },
    { type: 'p', children: '5' },
    { type: 'p', children: '6' }
  ]
}

没使用 Diff 时，需要执行 6 次 DOM 操作：

  • 3次 DOM 删除操作，卸载所有节点
  • 3次 DOM 新增操作，挂载所有节点

观察发现，新旧 vnode 都是 p 标签，只有 p 标签的子节点（文本节点）发生了变化，使用了 Diff 操作后，只需要执行 3 次 DOM 操作

  • 3次 DOM 文本节点更新操作，更新所有文本节点

代码简单实现如下：

function patchChildren(n1, n2, container) {
  if (typeof n2.children === 'string') {
    // 省略
  } else if (Array.isArray(n2.children)) {
    const oldChildren = n1.children;
    const newChildren = n2.children;
    const oldLen = oldChildren.length;
    const newLen = newChildren.length;
    // 两组子节点的公共长度，即两者中较短的那一组子节点的长度
    const commonLength = Math.min(oldLen, newLen);
    for (let i = 0; i < commonLength; i++) {
      // 新旧节点逐一diff
      patch(oldChildren[i], newChildren[i], container);
    }
    if (newLen > oldLen) {
      // 新节点数量大于旧节点，说明有新增节点
      for (let i = commonLength; i < newLen; i++) {
        patch(null, newChildren[i], container);
      }
    } else if (oldLen > newLen) {
      // 旧节点大于新节点数量，说明有需要卸载的旧节点
      for (let i = commonLength; i < oldLen; i++) {
        unmount(oldChildren[i]);
      }
    }
  } else {
    // 省略
  }
}

通过key复用节点

当一个节点的 key 和 tag 都相同的时候，我们就认为这个节点可以复用。节点可复用代表 DOM 元素可复用，不意味着就不需要更新（可能出现文本节点更新，或节点移动情况）

以下面新旧节点为例：

const oldVNode = {
  type: 'div',
  children: [
    { type: 'p', children: 'p-1', key: 1 },
    { type: 'p', children: 'p-2', key: 2 },
    { type: 'p', children: 'p-3', key: 3 }
  ]
}
const newVNode = {
  type: 'div',
  children: [
    { type: 'p', children: 'p-3', key: 3 },
    { type: 'p', children: 'p-1', key: 1 },
    { type: 'p', children: 'p-2', key: 2 }
  ]
}

对比新旧节点发现只需移动 key:1 和 key:2位置即可，相关 DOM 均可复用

diff 过程如下

  1. 取新节点第一个节点 p-3，key 为 3，在旧节点中可找到对应 key 值节点，索引为 2，节点复用，标记 lastIndex=2
  2. 取新节点第二个节点 p-1，key 为 1，在旧节点中可找到对应 key 值节点，索引为 0，节点复用，因索引 0<lastIndex=2，打破索引递增顺序，标记 p-1 需移动，无需更新 lastIndex 值
  3. 取新节点第三个节点 p-2，key 为 2，在旧节点中可找到对应 key 值节点，索引为 1，节点复用，因索引 1<lastIndex=2，打破索引递增顺序，标记 p-2 需移动，无需更新 lastIndex 值

实现代码如下：

function patchChildren(n1, n2, container) {
  if (typeof n2.children === 'string') {
    // 省略部分代码
  } else if (Array.isArray(n2.children)) {
    const oldChildren = n1.children
    const newChildren = n2.children
    // 用来存储寻找过程中遇到的最大索引值
    let lastIndex = 0
    for (let i = 0; i < newChildren.length; i++) {
      const newVNode = newChildren[i]
      for (let j = 0; j < oldChildren.length; j++) {
        const oldVNode = oldChildren[j]
        if (newVNode.key === oldVNode.key) {
          patch(oldVNode, newVNode, container)
          if (j < lastIndex) {
            // 如果当前找到的节点在旧 children 中的索引小于最大索引值 lastIndex
            // 说明该节点对应的真实 DOM 需要移动
            const prevVNode = newChildren[i - 1]
            // 如果 prevVNode 不存在，说明 newVNode 是第一个节点，不需要移动
            if (prevVNode) {
              const anchor = prevVNode.el.nextSibling
              // 将 newVNode 插入到 anchor 前面
              insert(newVNode.el, container, anchor)
            }
          } else {
            // 如果当前找到的节点在旧 children 中的索引不小于最大索引值，
            // 则更新 lastIndex 的值
            lastIndex = j
          }
          break // 这里需要 break
        }
      }
    }
  } else {
    // 省略部分代码
  }

// inset 函数
  insert(el, parent, anchor = null) {
    parent.insertBefore(el, anchor)
  }

双端Diff

双端 Diff 是 vue2 使用的， Diff 算法过程如下：

  1. 给四个指针赋值：newStartIdx=new p-4 newEndIdx=new p-3 oldStartIdx=old p-1 oldEndIdx=old p-4，初始状态如下图，开始第一轮 Diff
    • newStartIdx 和 oldStartIdx 对比：key 不同，不可复用，不处理
    • newEndIdx 和 oldEndIdx 对比：key 不同，不可复用，不处理
    • newEndIdx 和 oldStartIdx 对比：key 不同，不可复用，不处理
    • newStartIdx 和 oldEndIdx 对比：key 相同，复用，移动 p-4 DOM 节点，newStartIdx=p-2 oldEndIdx=p-3
  1. 第一轮 Diff 结束如下图，开始第二轮 Diff
    • newStartIdx 和 oldStartIdx 对比：key 不同，不可复用，不处理
    • newEndIdx 和 oldEndIdx 对比：key 相同，复用，位置相同，无需移动 newEndIdx=p-1 oldEndIdx=p-2
  1. 第二轮 Diff 结束如下图，开始第三轮 Diff
    • newStartIdx 和 oldStartIdx 对比：key 不同，不可复用，不处理
    • newEndIdx 和 oldEndIdx 对比：key 不同，不可复用，不处理
    • newEndIdx 和 oldStartIdx 对比：key 相同，复用，移动 p-1 DOM 节点，newEndIdx=p-2 oldStartIdx=p-2
  1. 第三轮 Diff 结束如下图，开始第四轮 Diff
    • newStartIdx 和 oldStartIdx 对比：key 相同，复用，无需移动，newEndIdx=p-4 oldStartIdx=p-3
  1. 第四轮 Diff 结束如下图，此时 newStartIdx > newEndIdx oldStartIdx > oldEndIdx跳出循环，结束 Diff

双端 Diff 算法对比简单 Diff 算法优势在于对于同样的更新场景，执行的 DOM 移动操作次数更少，如上述例子，使用双端 Diff 需移动 2 次 DOM，但使用简单 Diff 则需移动 3 次 DOM，如下图

快速Diff

vue3 使用快速 Diff 算法，快速 Diff 算法包含了预处理步骤，预处理是借鉴了纯文本 Diff 算法思路：

  1. 两段文本 Diff 之前，先进行全等比较，如果全等，无需进入 Diff：if (text1 === text2) return
  2. 预处理时还会处理两段文本相同的前后缀，对于相同的前后缀，无需进入 Diff

预处理节点过程：

  1. 建立新旧节点头索引从前向后遍历，直至遍历到不同节点，停止
  2. 建立新旧节点尾索引从后向前遍历，直至遍历到不同节点，停止
  1. 遍历完我们发现：
    • oldEnd < j，说明预处理过程中，所有旧节点都处理完
    • newEnd >= j，说明预处理过程中，存在未处理完的新节点，这些遗留的新节点视为新增节点，需要执行挂载处理
function patchKeyedChildren(n1, n2, container) {
  const newChildren = n2.children
  const oldChildren = n1.children
  // 处理相同的前置节点，索引 j 指向新旧两组子节点的开头
  let j = 0
  let oldVNode = oldChildren[j]
  let newVNode = newChildren[j]
  // while 循环向后遍历，直到遇到拥有不同 key 值的节点为止
  while (oldVNode.key === newVNode.key) {
    // 调用 patch 函数进行更新
    patch(oldVNode, newVNode, container)
    // 更新索引 j，让其递增
    j++
    oldVNode = oldChildren[j]
    newVNode = newChildren[j]
  }

  // 更新相同的后置节点，索引 oldEnd 指向旧的一组子节点的最后一个节点
  let oldEnd = oldChildren.length - 1
  // 索引 newEnd 指向新的一组子节点的最后一个节点
  let newEnd = newChildren.length - 1
  oldVNode = oldChildren[oldEnd]
  newVNode = newChildren[newEnd]
  // while 循环从后向前遍历，直到遇到拥有不同 key 值的节点为止
  while (oldVNode.key === newVNode.key) {
    // 调用 patch 函数进行更新
    patch(oldVNode, newVNode, container)
    // 递减 oldEnd 和 nextEnd
    oldEnd--
    newEnd--
    oldVNode = oldChildren[oldEnd]
    newVNode = newChildren[newEnd]
  }
}

如果回到预处理步骤 3，遍历完发现：

  • oldEnd >= j，说明预处理过程中，存在未处理完的旧节点，这些遗留的旧节点视为删除节点，需要执行卸载处理
  • newEnd < j，说明预处理过程中，所有新节点都处理完

如果回到预处理步骤 3，遍历完发现：

  • oldEnd >= j，说明预处理过程中，存在未处理完的旧节点
  • newEnd >= j，说明预处理过程中，存在未处理完的新节点
  • 新旧节点都有剩余，则需对这部分节点进行 Diff，我们可以构造一个 source 数组来存储预处理后剩余的节点，并且 source 中每个元素的初始值都是 -1，来辅助完成 DOM 移动操作

  • 填充完 source 数组值后，采用简单 Diff 逻辑对 source 数组中节点进行 Diff
